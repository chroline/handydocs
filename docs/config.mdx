There are 3 main parts to the HandyDocs configuration file:

1. **SEO** — search engine optimization; metadata about your site (site title, favicon, open graph images, etc.)
2. **Hero content** — styling and content for the hero
3. **Table of contents** — title of each content section

Each part will be explained more in-depth in their respective documentation sections.

The HandyDocs configuration file uses the YAML markup language and adheres to a strict structure, represented by the
following Typescript interface:

```typescript
interface ConfigFile {
  seo: any;

  hero: {
    style: {
      light: { bg: string; color: string; }
      dark?: { bg: string; color: string; }
    }
    logo: { text: string; }
    subheading: string;
    info_boxes: Array<{
      icon: string;
      title: string;
      url: string;
    }>
  }

  footer: {
    copyright: string;
    socials: {
      github?: string;
      twitter?: string;
      email?: string;
    };
    style: {
      light: { bg: string; color: string; }
      dark?: { bg: string; color: string; }
    }
  }

  toc: Record<string, string>;
}
```

<Alert status={"warning"}>

<AlertIcon />

Remember that the *actual* HandyDocs configuration file uses YAML. This Typescript interface is just to be used
for reference.

</Alert>

The un-edited boilerplate configuration will look like:

```yaml
seo:

hero:
  style:
    light:
      bg:
      color:
  logo:
    name:
  subheading:
  info_boxes:

toc:
```
